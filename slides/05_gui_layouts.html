Interfaces Gráficas (GUI)

  • Canal de interação entre o programa e seus usuários

  • Todas as ações do usuário são feitas por intermédio de uma GUI

  • Ganhou muita importância com o advento da tecnologia móvel

GUI ≠ UX

Nós focaremos em GUI!!

Android — GUI

  • No Android, as interface gráfica é composta por um catálogo de componentes

  • A montagem da interface é resultante da combinação desses componentes

  • Interface pode ser descrita em código ou XML

  • Existem basicamente dois tipos de componentes:

    • Layouts: lidam com o posicionamento dos elementos
    • Widgets: componentes com os quais o usuário pode visualizar e interagir

Propriedades dos Componentes

Todas os componentes possuem algumas propriedades em comum

Propriedade Descrição Valores
android:layout_width Define a largura que um componente irá ocupar match_parent
wrap_content
número (dp)

android:layout_margin Define a margem que um componente irá tomar de seu container número (dp)

O que findViewById faz?

A classe View

  • Classe base que representa todos os componentes visuais (Layouts e Widgets)

  • Qualquer componente herda a classe android.view.View

    • Obrigatório a implementção do método onDraw(canvas)

  • Widgets geralmente herdam diretamente a classe View

  • Layouts herdam View através da classe ViewGroup

GUI — Layouts

Layouts

São componentes que tem como propósito agrupar e organizar outros componentes na tela

Organização acontece de forma hierárquica

Todos os layouts devem herdar de android.view.ViewGroup

Principais Layouts

Layout Descrição
AbsoluteLayout Posiciona os componentes de forma fixa, utilizando coordenadas x e y. Deprecated!!
FrameLayout Tipo mais simples de Layout. Exibe um componente sobre outro, utilizando uma lógica de pilha.
LinearLayout Organiza os componentes linearmente, podendo ser configurado para exibí-los na horizontal ou vertical.
TableLayout Organiza os componentes no formato de uma tabela. Esse layout é uma derivação do LinearLayout.
RelativeLayout Organiza os componentes de forma relativa a outros. Por exemplo, posiciona componente A acima (a esquerda, direita ou embaixo) do componente B.

Outros layouts podem ser consultados aqui!!

FrameLayout

  • Implementação mais simples existente
  • Empilha um componente sobre outro no momento de sua exibição
    • Os últimos ficarão sobre os primeiros (FIFO)

LinearLayout

  • Um dos componentes de Layout mais comuns
  • Organiza os componentes linearmente
  • As linhas podem ser horizontais (padrão) ou verticais
    • Customizados pela propriedade android:orientation

LinearLayout

                        android:layout_width="match_parent"

android:layout_gravity configura o posicionamento do componente
android:weight configura o peso do componente

TableLayout

  • Layout que organiza a tela no formato de tabela
  • Útil para disposição de dados tabulares (Relatórios, formulários, etc.)
  • Linhas da tabela são representadas por android.widget.TableRow
    • Esse componente também é uma view, portando pode conter outros componentes

TableLayout

Possui dois atributos que podem alterar o formato da tabela

Atributo Descrição
android:stretchColumns Define quais colunas irão expandir-se ocupando o espaço disponível na tela. Similar ao atirbuto colspan do HTML.
android:shrinkColumns Define quais colunas serão sempre exibidas na tela.

TableLayout


            <?xml version="1.0" encoding="utf-8"?>
            <TableLayout ...
                android:layout_width="match_parent"
                android:layout_height="match_parent"
                android:stretchColumns="1" >
            
                <TableRow>
                    <TextView android:text="E-mail: " android:padding="3dip"/>
                    <EditText android:padding="3dip" android:inputType="textEmailAddress"/>
                </TableRow>
            
                <TableRow>
                    <TextView android:text="Senha: " android:padding="3dip"/>
                    <EditText android:inputType="textPassword"/>
                </TableRow>
            
                <TableRow android:gravity="right">
                    <Button android:text="Login"/>
                </TableRow>
            </TableLayout>

RelativeLayout

  • Posiciona os componentes em relação a um já existente
  • O posicionamento é feito com base em uma lista de atributos
  • Vantagem: Extremamente Flexível
  • Desvantagem: Fragilidade

                <RelativeLayout ... >

                    <TextView
                        android:layout_width="wrap_content"
                        android:layout_height="wrap_content"
                        android:layout_alignParentTop="true"
                        android:id="@+id/lblNome"
                        android:text="Nome:" />
                
                    <EditText
                        android:layout_width="match_parent"
                        android:layout_height="wrap_content"
                        android:layout_toRightOf="@+id/lblNome"
                        android:id="@+id/txtNome"/>
                
                    <TextView
                        android:layout_width="wrap_content"
                        android:layout_height="wrap_content"
                        android:layout_below="@+id/txtNome"
                        android:id="@+id/lblEmail"
                        android:text="E-mail:" />
                
                    <EditText
                        android:layout_width="match_parent"
                        android:layout_height="wrap_content"
                        android:layout_toRightOf="@+id/lblEmail"
                        android:layout_below="@id/txtNome"
                        android:id="@+id/txtEmail" />
                
                    <TextView
                        android:layout_width="match_parent"
                        android:layout_height="wrap_content"
                        android:layout_below="@id/txtEmail"
                        android:text="Sugestão"
                        android:id="@+id/lblSugestao"
                        android:layout_marginTop="15dip"/>
                
                    <EditText
                        android:layout_width="match_parent"
                        android:layout_height="match_parent"
                        android:layout_below="@id/lblSugestao"
                        android:layout_above="@id/btnEnviar"
                        android:id="@+id/txtSugestao"/>
                
                    <Button
                        android:layout_width="wrap_content"
                        android:layout_height="wrap_content"
                        android:layout_alignParentRight="true"
                        android:layout_alignParentBottom="true"
                        android:id="@+id/btnEnviar"
                        android:text="Enviar"/>
                
                    <Button
                        android:layout_width="wrap_content"
                        android:layout_height="wrap_content"
                        android:layout_toLeftOf="@+id/btnEnviar"
                        android:layout_below="@id/txtSugestao"
                        android:text="Limpar"/>
                    
                </RelativeLayout>

Dialogs

Pop-ups que aparecem na tela do usuário

Objetivos: Informar usuário ou requisitar a inserção de dados complexos

Herdam da classe android.app.Dialog

São criados diretamente em código

Todos os Dialogs são Layouts, porém com componentes e ações já pré-definidos!!

Dialogs

android.app.AlertDialog Caixa de diálogo genérica
android.app.ProgressDialog Exibe uma barra de progresso
android.app.TimePickerDialog Exibe um picker de tempo
android.widget.DatePickerDialog Exibe um picker de data

Dialogs


        //Criador de Dialog
        AlertDialog.Builder alertBuilder = new AlertDialog.Builder(this);

        // Define Parâmetros para o Dialog
        alertBuilder.setIcon(R.drawable.ic_launcher_foreground);
        alertBuilder.setTitle("Aviso!");
        alertBuilder.setMessage("Cuidado, isso pode acarretar em algo.");

        // Define o que acontece quando o usuário seleciona a opção positiva
        alertBuilder.setPositiveButton("Estou ciente!", new DialogInterface.OnClickListener() {
            @Override
            public void onClick(DialogInterface dialogInterface, int i) {
                Toast.makeText(getBaseContext(), "Fique por sua conta e risco", Toast.LENGTH_LONG).show();
            }
        });

        // Define o que acontece quando o usuário seleciona a opção negativa
        alertBuilder.setPositiveButton("Cancelar", new DialogInterface.OnClickListener() {
            @Override
            public void onClick(DialogInterface dialogInterface, int i) {
                Toast.makeText(getBaseContext(), "Muito bem!!", Toast.LENGTH_LONG).show();
            }
        });

        // cria o dialog e o exibe na tela
        AlertDialog dialog = alertBuilder.create();
        dialog.show();

GUI — Layouts & Adaptadores

Layouts & Adaptadores

  • Alguns Layouts podem ter seu conteúdo carregado dinamicamente
  • AdapterView é uma classe de layouts cujo seu conteúdo pode ser "injetado" durante execução do aplicativo
  • O carregamento das informações se dá através de um Adapter

O Padrão Adapter

Layouts & Adaptadores

Layouts que são implementados em conjunto com adaptadores:

ListView

GridView

ListView

Utilizada para exibir itens em lista sequencial

Os itens são recuperados a partir do adapter

O adapter também permite customizar a aparência dos itens da lista

Utilizando o ListView

  1. Implementar a classe Java que irá representar o item a ser exibido pela lista

  2. Implementar classe MyAdapter que herde de BaseAdapter

    • Você pode customizar o layout dos itens da lista através do método getView()

  3. "Ligar" o MyAdapter a ListView através do método setAdapter()

Exemplo ListView

1 — classe Java


        public class ProgrammingLanguage {

            private String name;
            private String author;
            private Integer year;
            
            public ProgrammingLanguage(String name, String author, Integer year) {
                this.name = name;
                this.author = author;
                this.year = year;
            }
            // ... getters e setters
        }

2 — Criando o Adaptador


        public class ProgrammingLanguageAdapter extends BaseAdapter {

            // lista que conterá a linguagens a serem exibidas
            private ArrayList<ProgrammingLanguage> languages; 
            private Context context;
        
            public ProgrammingLanguageAdapter(Context context) {
                this.context = context;
                languages = new ArrayList<>();
                //... carrega dados da lista
            }
            @Override
            public int getCount() {
                return this.languages.size();
            }
            @Override
            public Object getItem(int i) {
                return this.languages.get(i);
            }
            @Override
            public long getItemId(int i) {
                return i;
            }
            @Override
            public View getView(int i, View view, ViewGroup viewGroup) {
                ProgrammingLanguage language = this.languages.get(i);
        
                // cria o componente que será carregado na lista
                TextView lblLanguage = new TextView(this.context); 
                lblLanguage.setText(language.getName() + " -- " + language.getAuthor() + " -- " + language.getYear());
        
                return lblLanguage;
            }
        }
Sua classe deve herdade de BaseAdapter!!

3 — Ligando os pontos


            public class MainActivity extends Activity {

Extra — Modificando a Aparência da Lista

Para personalizar a aparência da lista é necessário:

  1. Criar o layout do item personalizado em um arquivo xml
  2. Importar dentro do método getView()

E.1 — Layout personalizado


        <?xml version="1.0" encoding="utf-8"?>
        <RelativeLayout xmlns:android="http://schemas.android.com/apk/res/android"
            android:layout_width="match_parent"
            android:layout_height="match_parent"
            android:padding="20dp">
                
            <TextView
                android:layout_width="wrap_content"
                android:layout_height="wrap_content"
                android:id="@+id/lbl_language_name"
                android:textSize="20dp"
                android:text="My Language"/>
        
            <TextView
                android:layout_width="wrap_content"
                android:layout_height="wrap_content"
                android:layout_alignParentEnd="true"
                android:id="@+id/lbl_language_year"
                android:textSize="20dp"
                android:text="1900"/>
        
            <TextView
                android:layout_marginTop="5dp"
                android:layout_width="match_parent"
                android:layout_height="wrap_content"
                android:layout_below="@id/lbl_language_name"
                android:id="@+id/lbl_language_author"
                android:text="A given Author"/>
        </RelativeLayout>
Não se esqueça de adicionar id aos componentes!!

E.2 — Alterando o getView()


        @Override
        public View getView(int i, View view, ViewGroup viewGroup) {
            ProgrammingLanguage currentLanguage = this.languages.get(i);
    
            // recupera a view do adapter que será customizada
            View newView = LayoutInflater.from(this.context).inflate(R.layout.adapter_languages, viewGroup, false);
    
            // recupera cada um dos campos do item
            TextView lblName = newView.findViewById(R.id.lbl_language_name);
            TextView lblAuthor = newView.findViewById(R.id.lbl_language_author);
            TextView lblYear = newView.findViewById(R.id.lbl_language_year);
    
            // define o valor de cada um dos campos
            lblName.setText(currentLanguage.getName());
            lblAuthor.setText(currentLanguage.getAuthor());
            lblYear.setText(currentLanguage.getYear().toString());
    
            return newView;
        }
